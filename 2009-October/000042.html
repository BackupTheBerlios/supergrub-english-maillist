I figured out why the CDs I created with GRUB 1.97 beta 3/4 couldn't read 
partition tables; the grub module names changed from 1.96 to 1.97. So 
while the grub.cfg tried to insmod pc, it needed to insmod part_msdos. I 
thought it was a problem with grub-mkrescue because the same grub.cfg 
worked when I used it from the same version of grub on my hard drive, not 
realizing that on my hard drive part_msdos was already embedded by 
grub-install :)

This is a patch to update all of the module names to those used in 1.97.

---
 menus/grub.cfg |   14 +++++++-------
 1 files changed, 7 insertions(+), 7 deletions(-)

diff --git a/menus/grub.cfg b/menus/grub.cfg
index e1a3004..62139e9 100644
--- a/menus/grub.cfg
+++ b/menus/grub.cfg
@@ -14,13 +14,13 @@
 # You should have received a copy of the GNU General Public License
 # along with GRUB.  If not, see <http://www.gnu.org/licenses/>.
 
-insmod biosdisk
-insmod acorn
-insmod amiga
-insmod apple
-insmod gpt
-insmod pc
-insmod sun
+insmod part_biosdisk
+insmod part_acorn
+insmod part_amiga
+insmod part_apple
+insmod part_gpt
+insmod part_msdos
+insmod part_sun
 
 # Set OSes names
