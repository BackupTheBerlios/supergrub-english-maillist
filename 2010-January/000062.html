    Stop changing $prefix before using configfile
    
     * Ubuntu's grub.cfg no longer fails when $prefix is not set to its own
     * /boot/grub, which was the main reason for changing $prefix in the
     * first place. Since changing $prefix also causes modules to be loaded
     * from the user's /boot/grub, which may not be compatible, I think
     * changing $prefix is currently more likely to cause problems than
     * solve them.
    
     * Still export $prefix ( and additionally export $root ) to work around
     * https://savannah.gnu.org/bugs/?func=detailitem&item_id=27710 .
     * Since this bug has been fixed in grub trunk, and the fix will be in
     * grub 1.98 when it releases, these redundant exports can be removed in
     * a future SGD release.
