Date:   Sat Mar 6 17:34:34 2010 -0800

    Add 'prepare-supergrub-environment.lua' and use $script_prefix from it
    
     * To allow for the same image to be used by multiple architectures
     * grub-mkrescue in grub 1.98 puts modules in different arch specific
     * directories, like "/boot/grub/i386-pc" while non arch specific files
     * like scripts stay in "/boot/grub/". Because of this we can't just do
     * things like "configfile $prefix/osdetect.lua" any more.
    
     * prepare-supergrub-environment.lua sets and exports $script_prefix
     * which is basically $prefix with the architecture removed. It also
     * sets $prefix_uuid, $prefix_device, $prefix_path, and $script_path.
     * These are so that when device names change ( for instance after
     * inserting ata.mod ) it will be easy to reset $prefix_device using
     * $prefix_uuid then from there update $prefix and $script_prefix.
