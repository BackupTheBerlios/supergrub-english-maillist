    Stop adding "Other OS" entries when no OS is found
    
     * Currently if no OS is detected on a partition, and it's a primary
     * partition, osdetect.lua adds a menu entry anyway titled "Other OS"
     * that just tries to chainload that partition. Most of the time if an
     * OS isn't found it's just a separate /home or swap partition and isn't
     * going to have a bootloader to chainload anyway. With this commit
     * osdetect will just return without adding an entry for partitions it
     * can't find an OS on.

    Add error messages to menu generating scripts
    
     * Right now if any of the scripts that dynamically add menu entries
     * fails to add any, for instance if cfgdetect.lua doesn't find any
     * grub.cfg files, the main menu just flashes. Instead these scripts
     * should print an error message explaining that no
     * {grub.cfg,core.img,iso,os} was found.
