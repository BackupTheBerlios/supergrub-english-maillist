Hi Adrian, hi all,

I nearly finished a new stable version of Boot-Repair.
As I promised to Adrian, I made it compatible with Debian, so that it can be
used it in Rescatux.
See: http://pix.toile-libre.org/upload/original/1312649224.png

Boot-Repair is already translated in 16 languages, and has much more
functionalities than Rescapp:
- reinstall and update GRUB,
- purge GRUB
- choice to place GRUB in MBR or partition
- restore generic Windows (XP, Vista, Seven) MBR   (current Rescapp does not
work with Vista/Seven)
- restore original MBR (required when computer has MBR-tatoo)
- auto detect separate /boot
- detect RAID
- compatible Debian, Ubuntu and derivatives
- can be used in live-CD, live-USB and installed session
- compatible whatever the architectures (32/64bits) of OS detected on the pc
- automatically solve several GRUB/Wubi bugs
- generate BootInfoScript report (automatically uses the very last script)
in 1 clic
- repair filesystems (linux AND windows)
- a nice and intuitive GUI
- ...
- most important of all : automatic selection of best choices to repair the
boot in 1 clic !


I think this is enough to consider adopting Boot-Repair in Rescatux ;)


If you are ok, I need your support for:
- translating remaining languages:
https://translations.launchpad.net/boot-repair/trunk
- voting the 3 bugs given in the 1st post of this thread:
http://ubuntuforums.org/showthread.php?p=10871917#post10871917
- do more tests (e.g. with raid and exotic configurations)
- make it compatible with non-Debian-based distributions
- putting Boot-repair in Debian repositories
- suggest improvements

Best regards
Yann
