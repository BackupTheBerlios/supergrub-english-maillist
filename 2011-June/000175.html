I am going to write a draft about Resclib.

Resclib description:
   It is a library geared towards (Gnu/Linux rescue).

Development:
    Although it might have more language hooks initially it will 
programmed with bash scripting.

boot-repair and resclib:
    resclib is going to be a package (initally Debian and Ubuntu ones) 
so boot-repair will have to depend on resclib package.

Rescatux and resclib (1st age):
    Most of the rescatux_lib.sh will be dropped into resclib.sh (or 
whatever name). So that rescatux_lib.sh sources resclib.sh and define 
some global functions that are specific to Rescatux.

Rescatux and resclib (2nd age):
    Rescapp will become a package and thus it will depend on resclib 
package.

BIS and resclib:
     Boot Info Script will also use resclib but due the special nature 
of it the resultant script will be one.

     This command reflects better what I mean:
     cat resclib.sh bisdo.sh > bis.sh

   BIS developers will only have to take care about bisdo.sh and the BIS 
users will only see bis.sh

System Rescue CD and resclib:
     Either in the form of a plain library, rescapp, boot-repair or in 
the form of a package they will be able to use it.
