Hi,

   As you might know per the main page I am rewriting Rescapp in QT 
thanks to Python.

   I explain the images.

   rescapp_python_1.png:
   =====================

   Well, that's it! This is the main menu. There are three columns so 
that we can :
   * Easily denote the right option thanks to its option (This is for 
not having problems with translations)
   * A button that opens a menu (It works right now) or selects an 
option (not implemented yet).
   * Know what the option is about

   That Main Menu button is to the return to this same screen, the main 
menu screen.

   At the bottom there is an integrated webbrowser which it is pointing 
at my local apache webserver. When you select an option its local 
documentation is supposed to be shown there. I suppose we will put 
videos there in the long term.


   rescapp_python_2.png:
   =====================

   That's a test I have done with the Support menu. You can see all its 
options.
   * Ignore that Run ls -l button I have removed it.
   * The integrated webbrowser is showing Spain's google.

   What's left in the UI
   ======================
   * Label that shows the selected option
   * Rescue! button so that it runs the option associated script (Only 
when there is a selected option)
   * Chat, Show log, Share log, Share log on forum will be permanent 
buttons. Probably in the right side of the screen.

   Other things needed
   ====================
   * Add rescapp.py to rescatux build process
   * Check that rescatux has all the required python libraries and 
binaries (add them to build process)
   * I'm not sure but I think that rescapp.sh defined some global 
variables before running the scripts. If that's true I will have to deal 
with that

So... This is all I want to develop till I officially release: Rescatux 0.30

   Rescatux 0.31
   =============
   * I want to implement thanks to python an screen that lets the user 
order their hard disks. (Far better than the current zenity way with n+1 
screens)
   * Maybe some other python screens for user interaction


   About the code
   ==============

   I won't update the code to GIT till I get the same functionality as 
the old rescapp in the UI. Doesn't matter if the UI is ugly or not. I 
might release a beta when that happens.

   Feedback
   ========

   What do you think?
