On Sat, Mar 17, 2012 at 1:31 AM, adrian15 <adrian15sgd at gmail.com> wrote:
>  It's quite annoying that the pause command (set pager=1 in grub2) does not
> work as expected inside menues.

"pager=1" is not a pause command, it's a pager command, and it works
the same way inside menu entries as outside them. "pager=1" means that
if more than a screenfull of text needs to be output, a pager should
be used. When less than a screenfull is printed no pager is needed and
thus no pager is used. This is similar to git's default behavior,
where "git status" will probably complete and return you to a prompt,
and "git log", which produces more than a screenfull of output, will
use a pager ("less"). If enough has changed that "git status" produces
more than a screenfull of output, then a pager will be used then also.

Since pager=1 is *not* a pause command, we need to use the actual
pause command ("sleep") to make sure that the text is viewable before
returning to the menu.
