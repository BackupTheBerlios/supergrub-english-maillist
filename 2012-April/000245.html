    Switch to GRUB's new gettext based translation system
    
     * Grub's new gettext based translation system uses the same syntax as
     * the bash translation system, $"string to be translated here". In
     * bash this syntax is not actually safe for various reasons, this is
     * *not* the case for grub's implementation which is perfectly safe.
    
     * Using this system allows more flexability for us as developers than
     * our current system allowing us to translate SG2D error messages
     * which would have been very difficult to translate previously, is
     * easier to use as a developer, and since it uses the standard gettext
     * system of po files should also be easier for translators wishing to
     * contribute translations to SG2D.
    
     * Since this is a new feature as of grub 2.0, this commit breaks
     * compatability with older versions of grub and so will not work with
     * grub 1.99 or older.
    
     * An added benefit is that when grub's own strings have available
     * translations, for example for the French and German languages, all of
     * grub's own strings (like the text below the menu, the outptut of "ls
     * -l", and grub error messages) will be translated as well!
