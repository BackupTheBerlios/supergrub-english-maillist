On Tue, Jan 24, 2012 at 2:38 PM, adrian15 <adrian15sgd at gmail.com> wrote:
> Rescatux 0.30 beta 4 is based on Super Grub2 Disk (before this commit).
>
> Does it mean that if I am going to boot it from Mac Book Pro (which has an
> EFI instead of a BIOS) is it going to complain about not being able to load
> biosdisk module?

No, Intel Macs have a BIOS implementation as well, and Rescatux 0.30
beta 4 only has grub-pc, not grub-efi. Since only a BIOS based
bootloader is available on the CD, an Intel Mac will use its BIOS
implementation to boot from the CD. Also even if the grub.cfg with
"insmod biosdisk" were to be used with grub-efi, an error message
would be printed, but it wouldn't hurt anything. In fact it would be
the same error message that's been printed in SG2D 1.98s1 because of
my mistake of "insmod part_biosdisk" that you fixed, that nobody had
ever even noticed :)
