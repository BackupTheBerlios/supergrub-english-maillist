I've added the "Luks support" feature request at the tracker:
https://forja.cenatic.es/tracker/index.php?func=detail&aid=1431&group_id=204&atid=930

It arises from an Spanish forum question:

http://www.supergrubdisk.org/forum/index.php?topic=753.0

Basically he says he wants LUKS support to be able to boot into its 
encrypted system from a usb.

What he has tried is to build is own SG2D with:
grub-mkrescue --modules="crypto" $default_options "$@"
(editing supergrub-mkrescue file)

And with a new "Enable GRUB2's LUKS support option which simply runs:
   insmod crypto

He says that Detect any os or Detect any Grub2 configuration do not seem 
to work as expected.

I haven't booted using LUKS myself so I'm not sure if it even makes 
sense. I suppose you have to write a password to even read the 
partition, isn't it?
