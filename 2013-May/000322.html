It detects Debian/Linux's grubx64.efi and boots it under EFI.  It
will also boot this system "normally" via the detected kernel/initrd
(no special kernel options required).

It detects Win8's efi files and bootmgfw.efi at least works to
boot it.   FTR the "Vista" entry created for the Win8 failed.   I
got someone else to check this for me in a VM they had handy
as no Win here.

When booted in bios mode it throws the error message if you
try and chainload an efi file.  It can of course still boot the
Debian/Linux system normally (kernel/initrd).

Or IOW afaik new stuff seems to work ok and I didn't find
anything old newly broken (I'm pretty sure the Win8 as "Vista"
failure is an old issue).
